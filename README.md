# 洛克王国 AI 对战工具

基于 Python Tkinter 的本地桌面工具，用于管理《洛克王国》精灵数据、维护对战阵容，并进行实战伤害推演。

## 功能

### 1. 精灵搜索与属性配置（主窗口）
- 从本地数据库（`data/all_pets_data.json`，支持 465+ 只精灵）按名称模糊搜索精灵
- 自动显示精灵的种族值、基础属性、元素属性、克制表、特性等信息
- 配置个体值（IV: 0/7/8/9/10）与性格修正系数（0.9/1.0/1.1/1.2）
- 实时计算并显示 6 项实战面板值（生命、物攻、魔攻、物防、魔防、速度）

### 2. 阵容管理
- 分别管理己方与对方阵容，两侧各支持多个命名阵容
- 将精灵（含数值配置、技能配置）添加到指定阵容
- 从阵容中删除精灵，支持删除整个阵容
- 阵容数据持久化保存到 `data/all_lineups.json`
- **阵容精灵详细数据**：保存时附带数值配置（基础属性、IV、性格系数）、克制表、特性、技能详情

### 3. 实战伤害推演（独立窗口）
- 支持"本人 vs 对方"双面板配置
- 精灵名称与技能名称输入时弹出候选列表（模糊搜索 + 键盘/鼠标选择）
- 技能参数配置：
  - 技能类型（物攻/魔攻/状态）
  - 技能属性（18 系 + 无）
  - 技能威力、本系加成（STAB，自动计算或手动覆盖）、属性克制倍率（自动计算或手动覆盖）
  - 增减益乘区、连击数、其他修正、技能耗能
- 伤害计算逻辑支持物攻/魔攻区分，支持能量系统（初始 10，技能消耗能量，聚能回复 5）
- 实战属性面板自动关联个体值与性格系数，支持特性"最好的伙伴"的实时倍率影响
- 当前支持特性：**最好的伙伴**（造成克制伤害后攻防速 +20%，回复 2 能量，可叠加）
- 支持从阵容直接加载精灵数据进行推演

### 4. 数据爬虫
- 异步爬虫（基于 httpx + BeautifulSoup），从 [Bwiki 洛克王国精灵图鉴](https://wiki.biligame.com/rocom/%E7%B2%BE%E7%81%B5%E5%9B%BE%E9%89%B4) 抓取精灵数据
- 支持并发限制（默认 3 并发），含被风控重试机制
- 抓取字段：基础属性、元素属性、种族值、技能列表（含等级/属性/消耗/类型/威力/描述）、血脉技能、技能石、克制/抵抗表、特性信息

### 5. 数据工具
- `check.py`：检查精灵数据完整性（基础属性、技能、克制表三项）
- `data_handle.py`：从原始数据中提取并去重特性库与技能库

## 运行环境

- Python 3.10+
- 需要 Tkinter（Linux 下可能需要 `python3-tk` 包）

### 安装依赖

```bash
pip install -r requirements.txt
```

### 启动程序

```bash
python3 main.py          # 启动主窗口
python3 damage.py        # 直接启动伤害推演窗口
python3 utils/crawler.py # 运行爬虫更新精灵数据库
```

依赖项：`httpx`、`beautifulsoup4`、`urllib3`

## 数据说明

- 精灵数据库默认从 `data/all_pets_data.json` 读取（465 只精灵，约 7MB）
- 阵容文件保存在 `data/all_lineups.json`
- 如果项目根目录下存在旧版 `all_lineups.json`，程序启动时会自动迁移到 `data/` 目录
- 运行时会在 `data/` 下生成 `battle_pet_cache.json` 供窗口间传递数据

### 精灵数据结构

每只精灵包含以下字段：
- `精灵ID`、`名字`、`详情连接`、`元素`（如 ["光"]）
- `种族值`（总和）、`基础属性`（生命/物攻/魔攻/物防/魔防/速度）
- `克制表`（克制/被克制/抵抗/被抵抗）
- `特性`（名称 + 效果描述）
- `精灵技能列表`、`血脉技能列表`、`可学技能石列表`

### 阵容数据结构

阵容按 `{"己方": {"阵容名": [...]}, "对方": {"阵容名": [...]}}` 组织。
每只精灵条目含：
- `名字`、`数值配置`（基础属性/IV/性格系数）
- `实战属性`（6 项面板值）、`克制表`、`特性`
- `技能配置`（技能详情列表）

## 核心公式

### 面板属性计算

```
生命 = round(1.7 × (种族 + 3 × IV) + 70) × 性格 + 100
其他属性 = round(1.1 × (种族 + 3 × IV) + 10) × 性格 + 50
```

### 伤害计算

```
最终伤害 = round(0.9 × 技能威力 × 攻击值 × 修正系数 / 防御值)
修正系数 = 本系加成 × 属性克制 × 增减益 × 连击数 × 其他修正
```

- 物攻伤害使用攻击方物攻 vs 防守方物防
- 魔攻伤害使用攻击方魔攻 vs 防守方魔防
- 状态技能不造成伤害
- 所有取整使用四舍五入

## 特性系统（扩展中）

当前实现：
- **最好的伙伴**：属性克制且造成伤害时触发，攻防速 +20%（可叠加），回复 2 能量

框架预留了 `trait_best_partner.py` 的模块化接口，可通过 `init_state()`、`current_multiplier()`、`on_after_damage()` 等标准化方法扩展新特性。
